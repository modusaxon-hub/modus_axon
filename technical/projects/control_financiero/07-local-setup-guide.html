Guía de lanzamiento y configuración de túnel · MODUS AXON

1. Ejecución Garantizada

Para evitar conflictos de puertos y asegurar que la aplicación siempre corra en el puerto 3000, utiliza el lanzador centralizado:

launch.bat
  • Estabilidad: El script detecta si el puerto está ocupado y lo libera automáticamente antes de iniciar.
  • Lanzamiento Dual: Inicia simultáneamente Next.js y el túnel de acceso público.

3. Control Total Supabase (CLI)

Para gestionar la base de datos y generar tipos de TypeScript automáticamente, asegúrate de estar logueado:

npx supabase login --token [TU_TOKEN]
npx supabase link --project-ref eqksdtluiuaeigbjdzrk

Generación de tipos (cada vez que cambie el esquema):

npx supabase gen types typescript --linked > src/lib/supabase/database.types.ts

4. Consideraciones Técnicas

  • Node.js: Asegúrate de tener instalada la versión 18 o superior.
  • Memoria: El servidor de desarrollo de Next.js requiere aproximadamente 200MB de RAM.
  • Seguridad: El túnel zrok es seguro, pero recuerda no exponer secretos en consola mientras el túnel esté activo.
