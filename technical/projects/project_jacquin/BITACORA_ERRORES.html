27/02/2026/E00013 Último VULN Resuelto VULN-007 — Sin Validación de Payload en Backend 24 Feb 2026 ›
Código Original VULN-007
Build Asociado 24022026SECFIX
Severidad Media
Fecha 24 Feb 2026
Riesgos identificados:
(a) Payload de decenas de MB podía enviarse libremente sin restricción de tamaño.
(b) Un array vacío [] pasaba la validación is_array() y borraba todos los programas.
(c) Arrays secuenciales como [{"title":"x"}] podían corromper la estructura del JSON.
Bug técnico descubierto: La detección de [] vacío con array_keys($data) === range(0, count($data)-1) fallaba porque range(0, -1) devuelve [0, -1] en PHP (no un array vacío). Bug descubierto durante la suite de tests y corregido inmediatamente.
Solución: Límite de 10 MB + rechazo de [] vacío usando empty() + rechazo de arrays secuenciales (estructura inválida).
27/02/2026/E00012 VULN Resuelto VULN-006 — Dos Copias Independientes del Endpoint de Escritura 24 Feb 2026 ›
Código Original VULN-006
Build Asociado 24022026SECFIX
Severidad Arquitectura
Archivos afectados:
jacquin_api/admin_save_programs_json.php (raíz, activo) y jacquin_api/public/admin_save_programs_json.php (duplicado).
Ambas con lógica ligeramente diferente (0777 vs 0755, mensajes distintos) pero apuntando al mismo programs.json. Duplica la superficie de ataque y complica el mantenimiento.
Solución: El endpoint canónico es el de la raíz (al que apunta ApiService). El duplicado en public/ devuelve HTTP 410 Gone indicando su obsolescencia.
27/02/2026/E00011 VULN Resuelto VULN-005 — Sin Cache Headers en get_programs_json.php 24 Feb 2026 ›
Código Original VULN-005
Build Asociado 24022026SECFIX
Severidad Performance
Archivo afectado: jacquin_api/get_programs_json.php.
Sin ningún header de caché, cada visita al home descargaba el JSON completo (838 KB al momento del hallazgo) desde InfinityFree sin posibilidad de caché del navegador.
Solución: Añadidos Cache-Control: public, max-age=300, ETag (MD5 del archivo) y Last-Modified.
Lógica de respuesta 304 Not Modified cuando el cliente ya tiene la versión actual. En visitas repetidas, el JSON no se descarga (0 bytes de transferencia).
27/02/2026/E00010 VULN Resuelto VULN-004 — Directorio data/ Creado con Permisos 0777 24 Feb 2026 ›
Código Original VULN-004
Build Asociado 24022026SECFIX
Severidad Baja-Media
Archivo afectado: admin_save_programs_json.php (raíz), línea 26: mkdir(dirname($file), 0777, true).
En hosting compartido (Linux), un directorio world-writable permite que cualquier proceso del servidor modifique programs.json.
Solución: 0777 → 0755. La copia en public/ ya usaba 0755 correctamente.
27/02/2026/E00009 VULN Resuelto VULN-003 — Sin Bloqueo de Archivo en Escrituras Concurrentes 24 Feb 2026 ›
Código Original VULN-003
Build Asociado 24022026SECFIX + 24022026STRESS
Severidad Moderada
Archivos afectados: Ambos endpoints de escritura usaban file_put_contents() sin flags.
En escritura simultánea de dos admins, los bytes podían intercalarse produciendo un JSON corrupto que dejaba Programs.jsx en blanco (el JSON.parse fallaría silenciosamente).
Solución: Añadido flag LOCK_EX en todas las llamadas file_put_contents() de ambos archivos.
Verificado por test de 5 procesos paralelos (Build: 24022026STRESS).
27/02/2026/E00008 VULN Resuelto VULN-002 — Imágenes Base64 Embebidas en programs.json (838 KB) 24 Feb 2026 ›
Código Original VULN-002
Build Asociado 24022026SECFIX
Severidad Alta — Performance
Evidencia real: program_1768517168737 contenía una imagen Base64 de 844,922 caracteres. El JSON completo pesaba 858,635 bytes (838 KB).
Cada visita al home descargaba esos 838 KB antes de renderizar "Nuestros Programas".
Proyección máxima con 9 programas × 3 MB: JSON de ~36 MB.
Solución backend: El endpoint detecta automáticamente cualquier campo image que empiece con data:image/, decodifica el Base64, valida tipo MIME (jpg/png/webp/gif), valida tamaño (máx 5 MB), guarda en public/uploads/programs/ con PathHelper, y sustituye el campo por la ruta relativa (<30 chars).

Migración ejecutada: programs.json pasó de 858 KB → 5 KB (−99.4%). Imagen guardada como archivo PNG físico (618 KB).
Frontend: Programs.jsx actualizado con resolveImageUrl() para manejar rutas public/uploads/ vía ApiService.BASE_URL.
27/02/2026/E00007 VULN Resuelto VULN-001 — Sin Autenticación en Endpoint de Escritura de Programas 24 Feb 2026 ›
Código Original VULN-001
Build Asociado 24022026SECFIX
Severidad Crítica
CVSS Estimado 9.1 Crítico
Archivos afectados: jacquin_api/admin_save_programs_json.php (raíz) y jacquin_api/public/admin_save_programs_json.php (copia).
Ambos carecían de session_start() y verificación de rol. Cualquier actor externo podía enviar un POST anónimo y sobrescribir o vaciar programs.json, borrando la sección pública instantáneamente.
Solución: Añadido session_start() + guard $_SESSION['user']['id_rol'] != 1 → HTTP 403 en ambos archivos.
El endpoint public/ marcado además como deprecado (HTTP 410).
ApiService.saveProgramsJson() actualizado con credentials: 'include' (legacy + React) para transmitir la cookie de sesión PHP.
