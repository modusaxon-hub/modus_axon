24/02/2026/B00008 Reconexión Flujo de Inscripción — Modal Programas 24 Feb 2026 24022026ENROLL ›
Instrucción Original
"Audita la lógica y el flujo perdido de la sección de nuestros programas. Cuando se cliquea en un programa ofertado, no se despliega el modal. Maneja todos los casos: logueado, no logueado, no registrado, intento de inscripción y recuperación del hilo del proceso."
Causa Raíz Identificada
Conflicto de paradigma: Vanilla JS vs. React SPA. programs_carousel.js (472 líneas) implementa modal/inscripción en Vanilla JS pero nunca fue incluido en index.html de la SPA. window.openProgramModal era undefined silenciosamente.
  • Bug secundario: navigate('/#programas') no dispara DOMContentLoaded en SPA → recovery de intent nunca ocurría.
  • Field mismatch: programs_carousel.js:294 usaba c.name pero la API retorna c.course_name.
Solución Implementada
  • ProgramModal.jsx (CREADO): Modal React con imagen, título, descripción, beneficios, botón "Inscribirme".
  • ScheduleModal.jsx (CREADO): Carga cursos, match por nombre, selección de horario, requestEnrollment().
  • AuthPromptModal.jsx (CREADO): Overlay para visitantes. Guarda intent en sessionStorage antes de redirigir.
  • Programs.jsx (REFACTORIZADO): State selectedKey + modalStep. Recovery via useLocation().
  • Login.jsx (CORREGIDO): navigate('/', { state: { recoverEnrollment: true } }).
