1. Introducción

Este documento técnico describe la implementación de estándares de calidad en el desarrollo de Project Jacquin. Se detalla cómo las decisiones de arquitectura y codificación cumplen con los requisitos de la norma ISO/IEC 25000.

2. Modelo de Calidad (ISO/IEC 25000)

Se han priorizado los siguientes atributos:

  • Usabilidad: Interfaces intuitivas y limpias (Brandbook V2).
  • Eficiencia de Desempeño: Eliminación del problema N+1 en bases de datos mediante consultas optimizadas (`GROUP_CONCAT`).
  • Portabilidad: Diseño responsivo (Mobile-First) con CSS Vanilla.
  • Mantenibilidad: Arquitectura desacoplada (Frontend vs Backend API).

3. Buenas Prácticas Implementadas

3.1. Estructura Semántica

Uso estricto de HTML5 para SEO y accesibilidad.

<header> <nav role="navigation">...</nav> </header> <main> <section aria-label="Cursos Disponibles">...</section> </main>

3.2. Optimización Técnica

  • Lazy Loading: Atributo loading="lazy" en iframes y recursos pesados.
  • Seguridad: Cookies con flags `HttpOnly` y `Secure` para prevenir XSS.
  • Zrok Compatibility: Headers CORS configurados para operar en túneles seguros.
