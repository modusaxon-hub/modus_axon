1. Introducción y Concepto

"La Academia JACQUIN es una sinfonía de ingeniería donde el diseño premium y la lógica algorítmica trabajan en perfecta armonía administrativa."

Este documento consolida la totalidad del conocimiento técnico del Project Jacquin. El sistema ha sido diseñado como una Oficina Virtual Académica que centraliza la gestión de usuarios, inscripciones, horarios y calificaciones, eliminando la redundancia y garantizando la integridad de los datos mediante un motor de conflictos algorítmico.

2. Ingeniería de Requerimientos

Funcional

RF-01: Autenticación Robusta

Acceso diferenciado por perfiles (Admin, Docente, Estudiante) con sesiones controladas de 8 horas y manejo de timeouts.

Funcional

RF-02: Inscripción Inteligente

El sistema valida el aforo (`max_students`) y los conflictos de horario antes de permitir una matrícula.

Calidad

RNF-01: Hash BCRYPT

Las contraseñas nunca se almacenan en texto plano. Se utiliza el algoritmo BCRYPT con un coste computacional adecuado.

Calidad

RNF-02: API RESTful

Separación total entre Frontend (HTML/JS) y Backend (PHP/JSON), permitiendo futura expansión a aplicaciones móviles.

Componentes Clave

  • Config (`config/db.php`): Singleton de conexión a base de datos usando PDO.
  • Middleware (`middleware/auth_helper.php`): Interceptor que valida sesiones y roles antes de ejecutar lógica de negocio.

5. Lógica de Negocio Crítica

Algoritmo de Detección de Conflictos

Para evitar que un estudiante se inscriba en dos clases simultáneas, o que un profesor sea asignado doblemente, se utiliza la **Intersección de Intervalos**:


// Pseudocódigo de Validación
function hasConflict($newStart, $newEnd, $existingSchedules) {
    foreach ($existingSchedules as $schedule) {
        // Fórmula de Superposición
        if ($newStart < $schedule->end && $newEnd > $schedule->start) {
            return true; // Conflicto detectado
        }
    }
    return false;
}

Esta lógica reside en helpers/conflict_helper.php.

Proceso de Validación de Cupos

  1. Endpoint request_enrollment.php recibe la solicitud.
  2. Consulta SELECT count(*) de inscripciones activas del curso.
  3. Compara contra max_students del curso.
  4. Si count >= max, retorna 403 Forbidden: "Cupos Agotados".

7. Anexo: Explicación Didáctica

Explicación simplificada para stakeholders no técnicos (Directivos, Jurados).

Frontend (La Cara)

Es lo que el usuario ve. Botones, formularios y colores. Fue diseñado para ser intuitivo y responsive.

Backend (El Cerebro)

Es quien toma las decisiones. Calcula si hay espacio en el salón y guarda las notas de forma segura.

Flujo de "La Recepcionista Virtual"

  1. El alumno llega a la ventanilla virtual (Web).
  2. Pide entrar a una clase (Inscripción).
  3. La recepcionista (API) revisa el libro de reservas (Base de Datos).
  4. Si hay espacio y no choca con otra clase, le da el carnet (OK).

8. Guía de Despliegue (InfinityFree)

  1. Subir el contenido de web_page/pages/public/ a la raíz de /htdocs/.
  2. Subir la carpeta jacquin_api/ como una subcarpeta en /htdocs/api/.
  3. Subir la carpeta uploads/ a la raíz para que el sistema de gestión de archivos funcione.
  4. Configurar PathHelper.php para que apunte a la ruta raíz del hosting.
