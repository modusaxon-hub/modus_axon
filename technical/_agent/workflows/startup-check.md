// turbo-all
Para asegurar que el desarrollo esté alineado con la visión del usuario y no se pierdan nuevas ideas, sigue estos pasos al inicio de cada sesión o cuando se retome un proyecto:

1. Busca el archivo `01-idea-de-negocio.txt` en la carpeta de documentación técnica del proyecto (ej: `technical/projects/[nombre]/01-idea-de-negocio.txt`).
2. Lee el contenido completo del archivo usando `view_file`.
3. Identifica nuevas anotaciones o párrafos agregados por el usuario (normalmente al final del archivo o marcados con viñetas/notas).
4. Compara las ideas encontradas con los documentos de arquitectura (`product-vision.md`, `technical-spec.md`, `design-spec.md`).
5. Si detectas una nueva idea o instrucción:
   - Resúmela al usuario para confirmar entendimiento.
   - Propón los cambios necesarios en la documentación técnica y/o en el código.
   - Ejecuta la implementación si se recibe la aprobación.
6. Actualiza el documento `00-doc-index.md` si se generan nuevos documentos derivados de estas ideas.
